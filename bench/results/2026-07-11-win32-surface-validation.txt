DEFECTS FOUND AND FIXED
-----------------------

The first surface run rejected the Windows monotonic timer because signed
`LONGLONG` counter arithmetic was multiplied by an unsigned constant before the
explicit conversion. The implementation now:

    - checks QueryPerformanceFrequency success
    - rejects a zero or negative frequency
    - checks QueryPerformanceCounter success
    - rejects a negative counter
    - converts the counter before unsigned nanosecond arithmetic

The MSVC C configuration also removed `/permissive-`. DEADFLASH is a C-only
project; using a C++ conformance switch could produce an ignored-option warning
that becomes a hard failure under `/WX`.

BOUNDARY
--------

This is a compile-time API and conditional-source surface gate. The local stub
headers model only the Win32 APIs used by DEADFLASH. CL driver mode exercises
MSVC-compatible command-line and warning behavior while still using the Linux
host target for C runtime headers.

It does not link Windows libraries, execute Windows code, validate Windows ABI
layout, replace the Windows SDK, or count as an MSVC pass.

Actual MSVC build/test and sacrificial Windows physical-device qualification
remain mandatory release gates.
